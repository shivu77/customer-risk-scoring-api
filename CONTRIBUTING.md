# Contributing Guidelines

## Branch Naming
- Use `feature/<short-topic>`, `fix/<short-topic>`, or `docs/<short-topic>`

## Commit Messages
- Use imperative tone: `Add customer create endpoint`
- Include scope and brief description

## Pull Request Process
- Keep PRs focused and small
- Link related issues if available
- Ensure tests pass and lint is clean
- Provide a short description of changes
